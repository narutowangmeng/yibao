# 医保管理系统 - 角色与职责分工

## 一、三层组织架构

### 第一层：决策层（局领导）
- **医保局领导** (bureau_leader)
  - 统筹全局工作
  - 审批重大决策
  - 查看全局数据报表

### 第二层：管理层（三个业务司）
负责制定标准、政策、基础数据管理

| 部门 | 角色 | 核心职责 |
|------|------|----------|
| 待遇保障司 | treatment_director | 制定筹资政策、待遇标准、险种配置 |
| 基金监管司 | fund_supervisor | 制定监管规则、飞行检查、违规查处 |
| 医药服务管理司 | medical_service_director | 制定医药价格、招标采购、机构准入标准 |

### 第三层：执行层（经办人员）
负责具体业务办理，使用第二层制定的标准和数据

| 角色 | 职责 | 使用数据 |
|------|------|----------|
| 经办人员 (operator) | 参保登记、信息变更、日常业务 | 使用待遇保障司配置的险种、缴费标准 |
| 审核人员 (auditor) | 费用审核、稽核检查 | 使用待遇保障司的待遇政策、基金监管司的监管规则 |
| 财务人员 (finance_staff) | 基金结算、拨付 | 使用待遇保障司的缴费数据、基金监管司的监控数据 |
| 医疗机构管理员 (institution_admin) | 机构业务、医保结算申请 | 使用医药服务管理司的目录、价格标准 |
| 参保单位管理员 (employer_admin) | 单位参保、员工缴费 | 使用待遇保障司的险种、缴费标准 |
| 参保人员 (insured_person) | 个人查询、业务办理 | 查看个人参保、报销信息 |

## 二、数据流向

```
管理层（制定标准） → 执行层（使用标准办理业务） → 数据回流（统计监控）

待遇保障司          → 经办人员/财务人员           → 基金监管司（监控）
- 险种配置          → 参保登记使用
- 缴费标准          → 缴费核定使用
- 待遇政策          → 报销审核使用

医药服务管理司      → 医疗机构/审核人员           → 基金监管司（监控）
- 医保目录          → 费用结算使用
- 医疗服务价格      → 收费依据
- 支付方式标准      → 结算规则

基金监管司          → 全系统
- 监管规则          → 智能监控预警
- 信用评价          → 机构准入参考
```

## 三、功能模块划分

### 管理层功能（政策/标准制定）

**待遇保障司**
- 险种管理 - 配置险种类型、缴费标准、待遇参数
- 待遇政策 - 制定报销比例、起付线、封顶线
- 筹资政策 - 制定缴费基数、费率
- 医保目录管理 - 目录调整申请（与医药服务司协作）
- 长期护理保险政策
- 异地就医政策

**基金监管司**
- 监管规则配置 - 风控模型、预警规则
- 飞行检查计划
- 违规查处案件
- 信用评价体系
- 举报投诉处理

**医药服务管理司**
- 医疗机构准入标准
- 医保目录制定（药品、诊疗、耗材）
- 医疗服务价格制定
- 招标采购管理
- 支付方式改革标准（DRG/DIP分组）

### 执行层功能（业务办理）

**经办人员工作台**
- 参保登记（使用待遇保障司配置的险种）
- 信息变更
- 关系转移
- 缴费核定（使用待遇保障司的缴费标准）

**审核人员工作台**
- 费用报销审核（使用待遇保障司的待遇政策）
- 稽核检查（使用基金监管司的监管规则）
- 异常数据核查

**财务人员工作台**
- 基金收支管理
- 结算拨付
- 财务报表

**医疗机构工作台**
- 费用结算申请（使用医药服务司的目录和价格）
- 医保目录查询
- 结算单管理

**参保单位工作台**
- 单位参保管理
- 员工增减员
- 缴费申报

**个人服务大厅**
- 参保信息查询
- 报销进度查询
- 缴费记录查询
- 异地就医备案

## 五、权限控制原则

1. **管理层** - 查看全局数据、制定标准、审批
2. **执行层** - 办理具体业务、查看自己经办的数据
3. **外部用户** - 查看自己的数据、提交申请
